# FIX_GUIDE

## 修复记录

### 修复 #1：侧栏「商品分类」按钮可见性 / 可发现性问题

**日期：** 2026-06-12
**页面：** `frontend/src/pages/ShopDetailPage.tsx`

---

#### 问题描述

用户进入店铺详情页后，无法注意到左侧的「全部商品 / 上新推荐 / 本周热销 / 优惠专区」筛选按钮，导致筛选/排序功能形同虚设。

#### 问题根因分析

1. **无区域标题**：侧栏是一个孤立的卡片容器，没有任何标题或说明文字告诉用户这是"商品分类/筛选"区域。
2. **字体过小**：按钮文字使用 `text-xs`（约 12px），在 1080p 以上屏幕非常难阅读。
3. **无图标辅助识别**：纯文字按钮缺乏视觉锚点，用户扫一眼页面时很难捕捉到。
4. **非激活态对比度低**：非选中按钮仅使用 `text-secondary-600` 灰色文字 + 无底色，视觉上和页面背景融为一体。
5. **视觉层级过弱**：按钮之间缺乏明显的分组和层次，看起来不像一组可交互控件。

#### 修复方案

| 修复点 | 修改前 | 修改后 |
|---|---|---|
| 区域标题 | 无 | 新增「商品分类」标题 + 说明文字「快速筛选找到心仪商品」 |
| 图标 | 无 | 每个按钮配对应 lucide-react 图标（Package / Sparkles / Flame / Percent） |
| 字体 | `text-xs` font-black | `text-sm` font-bold |
| 非激活态样式 | 文字灰色，无底色 | `bg-secondary-50` 浅灰底色 + `text-secondary-700` 深灰文字 + hover 边框高亮 |
| 激活态样式 | 纯 `bg-primary` | 渐变色 `from-primary to-primary-600` + 阴影 `shadow-primary/30` + 轻微放大 `scale-[1.02]` + 右侧圆点指示 |
| 按钮圆角 | `rounded-xl` | `rounded-2xl` 更大圆角，更符合整体设计风格 |
| 分类数据 | 内联字符串数组 | 抽离为 `CATEGORIES` 常量，含 name + icon 字段 |
| 侧栏宽度 | `lg:w-64` | `lg:w-72`，给图标和文字留出舒适空间 |

#### 修改文件

- `frontend/src/pages/ShopDetailPage.tsx`

#### 新增依赖

无。使用项目中已有的 `lucide-react` 图标库。
